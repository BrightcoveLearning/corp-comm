Step-by-Step: Live Streaming with Open Broadcaster Software (OBS)

The purpose of this Quick Start is to show you how to create a live event in Brightcove Engage and stream it in a Mobile App Experience. This topic will use Open Broadcaster Software (OBS) to broadcast a live event.

OBS is open source software for video recording and live streaming and can be downloaded here. Other encoders can also be used.

Note: For a version of this topic that uses the Telestream Wirecast encoder, see Step-by-Step: Using the Live Module with Telestream Wirecast.

After completing this Quick Start, you should be able to:

  • Create a live event
  • Configure OBS for a live event
  • Stream a live event in a Mobile App Experience

Prerequisites

  • A camera to broadcast the event
  • You must have your own encoding software; this topic uses OBS Studio which can be downloaded for free

Limitations

  • Your encoder must output an RTMP stream
  • Live broadcasts will experience a 20-30 second delay
  • Streaming must begin within 30 minutes of the event start
  • For users using Internet Explorer, version 10 or later is required

Steps to broadcast a live event

To stream a live event, you will complete the following tasks:

  • Create a new live event
  • Configure your encoder
  • Create a new Mobile App Experience
  • Start the live stream

Creating a new live event

  1. Login to Brightcove Engage.
  2. Click Live in the navigation header.
  3. Click Create Event to create a new event.
  4. Enter an Event Name.
  5. For the Tags, enter live event. We'll use this tag to search for the event when creating a collection later in this topic.
  6. Select Convert event to video asset when complete. This will save the event as a VOD video that will appear on the Videos page.
  7. In the STREAM OPTIONS section, select the Region closest to your location.
  8. Click Create Event.

The Live module will return a Stream Endpoint (RTMP URL) and Stream Name. These values will be used by your encoder.

Configuring your encoder

In this section, we will configure and use the OBS Studio encoder to support a live streaming event. OBS Studio software is a desktop application that captures input from your camera and produces a stream that can be delivered by a CDN. Note that there are hardware and other software based encoding solutions available that may be better suited for delivering your live streaming event. You can download the OBS Studio software here.

Note: You should have the OBS software installed and a camera connected to your computer before proceeding
Note: This topic uses a DSLR connected to a laptop using the Blackmagic Design UltraStudio Mini Recorder.
Note: OBS offers the ability to record a local copy of the stream. By choosing to do this, you will have a local copy of the stream that you can create clips from in case any networking issues occur during the broadcast of the event. For information on how to record a local file, see the OBS documentation.
  1. Open OBS Studio.
  2. In the Sources section, click + to add a new source. Select a Source from the list. This Quick Start is using a DSLR connected to a laptop using the Blackmagic Design UltraStudio Mini Recorder, so Blackmagic Device is selected.
  3. Enter a name for the new source and click OK.
  4. Select your camera from the Device list. A preview should appear in the properties dialog. Click OK.
  5. In the Controls section, click Settings.
  6. Click Stream in the left side of the Settings dialog.
  7. For the Service select Custom....
  8. For the Server, use the Stream URL provided by the Live module.
  9. For the Stream key, use alive.
  10. Click OK to save the settings.

Don't start streaming, we will do that after the Mobile App is created.

Creating a Mobile App Experience

In this section, we will create a simple Mobile App Experience and configure it so the featured video (the video that appears when the app is opened) will be a live event video.

Note: This section will not go through all the steps to create a "full featured" Mobile App Experience. For a step-by-step guide to creating a Mobile App Experience, see Step-by-Step: Creating and Publishing a Mobile App Experience.

  1. Click Projects in the navigation header.
  2. Click New Experience.
  3. Locate the Mobile App template and click Choose.
  4. Enter Mobile App Live Experience for the Experience Title.
  5. Click Create Experience.
  6. Click the Add Logo link on the experience preview.
  7. Click browse and then browse and select an image to use in the header.
  8. Click Save.
  9. Verify that the image appears in the experience preview.
  10. Click VIDEOS in the left navigation.
  11. Name the first collection Featured Video and then press enter.
  12. Click the collection name to open the collection editor.
  13. Select Custom for the criteria.
  14. In the Keywords field, enter live event and press enter.
  15. Verify that the Sample Live Event appears in the Video Collection list.
  16. Click Save.
  17. (Optional). Create additional collections.
  18. Publish the experience.
  19. Open the Brightcove Engage App on your device and enter the Access Key.
  20. The live event should appear when the app opens with UPCOMING EVENT listed next to the event name.

 

Starting the live stream

In this section, we will start the live stream and confirm that it appears in the Brightcove Engage app.

  1. Return to OBS Studio.
  2. In the Controls section, click Start Streaming. The live stream should begin.
  3. Return to the Engage app on your device. After a brief delay, the LIVE NOW should appear next to the event name.
  4. Click Play to play the live stream.

Ending the live stream

To end the live stream, follow these steps.

  1. Click Live in navigation header.
  2. Click on the event name to open the Control Room page.
  3. Click Stop Streaming.
  4. Return to OBS and click Stop Streaming.

Troubleshooting live stream issues

If you need further help getting your live event to work, you can contact us. To make sure you get the fastest response possible, below is a list of what support will need to solve the problem.

  • The specific symptoms the stream is having. For example, does it not play at all or does it stutter or freeze?
  • Whether this stream worked correctly in the past
  • The entry point URL you are using in your encoder
  • The encoding software and hardware are you using
  • The URL to the player to which you have published the live event
  • The video ID of your live asset in Video Cloud Studio
  • The results of a trace-route from your encoder to the publishing point host
