Step-by-Step: Creating and Publishing a Mobile App Experience

This topic provides a set of steps you can follow to create and publish a Mobile App experience using Brightcove Engage.

After completing this topic, you should be able to:

  • Use Brightcove Engage to create a new Mobile App experience
  • Add videos to an experience
  • Customize an experience
  • Preview and publish an experience
  • Install the Brightcove Engage app on a device and access an experience
Note: For a step-by-step guide to creating a Web experience, see Step-by-Step: Creating and Publishing a Web Experience.

Audience

Anyone who needs to create native mobile apps to display corporate video content.

Prerequisites

  • Access to Brightcove Engage

Before you get started

This step-by-step guide will use the videos that are included with the Brightcove Engage trial account. Feel free to upload and use your own videos.

  1. If you are using your own videos, you should create several playlists to organize the videos.
  2. Create one playlist called Featured Video that contains one video.

Installing the Brightcove Engage app

The Brightcove Engage mobile app is required to view published Mobile App experiences. Follow these steps to install the app.

  1. Open the App Store (iOS) or the Google Play Store (Android) on your device.
  2. Search for the Brightcove Engage app.
  3. Download and install the app on your device.

The app will be used later after publishing a Mobile App experience

Creating a Mobile App experience

To create a Mobile App experience using Brightcove Engage, you will complete the following tasks:

  • Create a new experience
  • Edit the site details
  • Add a logo to the main page
  • Create collections and add videos to the collections
  • Preview the experience
  • Customize the style
  • Publish the experience
  • Access the experience on a device

Creating a new experience

To create a new Mobile App experience, follow these steps.

  1. Login to Brightcove Engage.
  2. Click Projects in the top navigation and then click the Experiences link.
  3. If any experiences have been created, they will be listed. Click New Experience. If no experiences have been created, you will be directed to the Create Experience page.
  4. Locate the Mobile App template and click Choose.
    Note: Once you select a template it can't be changed. For more information templates, see the Overview of Experience Templates topic.
  5. Enter Sample Mobile App for the Experience Title. For the Pre-Populate experience? setting, select No thanks.
  6. Click Create Experience.

The Experience Editor will open. The Experience Editor is used to customize the experience and provides the ability to add videos, choose styles and configure other settings. All of the customization settings are accessed using the left navigation menu.

Editing the site details

The experience title, description and language settings can be edited using the Experience Editor.

  1. Click OVERVIEW in the left navigation.
  2. For the Experience Description, add Sample Mobile App Experience.
  3. Click Save.

Adding a logo to the main page

Each experience is comprised of a set of page templates. The page templates that are provided depend upon the template that was selected when the experience was first created. The Mobile App template provides Home, Collections, Video Details, Search and Profile page templates. These pages can be seen in the left margin as part of the PAGES menu.

The Mobile App template provides the ability to add a logo to the Home page. For information on adding a logo, see Adding a Logo to a Mobile App Experience.

To add a logo to the Home page, follow these steps.

  1. Click PAGES in the left navigation.
  2. Click the Home page in the left margin.
  3. Click on the Add Logo link.
  4. Click browse and then browse and select an image to use in the header.
  5. Click Save.
  6. Verify that the image appears in the experience preview.

Creating collections and adding videos to the collections

Using the Experience Editor, you can select which videos to display in the experience. When creating experiences, videos are organized into collections. Collections are created based upon video metadata and can include:

  • Playlists
  • Keywords
  • Custom metadata
  • Tags

When creating Mobile App experiences, the first video in the first collection is used as the featured video. The featured video appears at the top of the application.

When creating collections for Mobile App experiences, Brightcove recommends creating a Featured Video collection as the first collection. By assigning a manual playlist to the Featured Video collection, the featured video can be easily be changed by editing the manual playlist using the Videos page. This also allows you to make changes to the featured video without needing to edit and republish the experience.

For more information on creating collections and adding videos to an experience, see Adding Videos to an Experience.

To add videos to the experience, follow these steps.

  1. Click VIDEOS in the left navigation.
  2. Name the first collection Featured Video and then press enter.
  3. Click the collection name to open the collection editor.
  4. Select Playlist for the criteria.
  5. Click the Featured Video playlist. The video(s) in that playlist will appear in the videos list. Click Save.
    Note: The Featured Video playlist was created in the Before you get started section above.
    Note: When using a playlist video assignment, the order of the videos is based upon the playlist settings that were set in the Videos page.
  6. Create another collection named Welcome Aboard! using the Welcome Aboard! playlist as the criteria.
  7. Create another collection named CEO Updates using the CEO Updates playlist as the criteria.
  8. Confirm that 3 collections have been created.
    Note: You can change the order of the collections by clicking and dragging the collection to a new position.

Previewing the experience

Previewing a Mobile App experience lets you see how it will appear on mobile devices. A quick preview can be done by clicking on the page templates that appear under the PAGES navigation item. Note that when using this preview option, the videos will not play but you can get a view of the layout.

The experience can be previewed in a new browser tab or a shareable link can be generated so others can preview. For more information on previewing and publishing a Mobile App experience, see Previewing and Publishing a Mobile App Experience.

To preview the experience, follow these steps.

  1. While editing the site using the Experience Editor, click Preview and then click Responsive Preview.
  2. The preview will open in a new browser tab.
  3. Close the site preview tab.

Customizing the style

The Style settings can be used to customize the colors that are used on the experience. For more information on changing the style for Mobile App experiences, see Styling Mobile App Experiences.

To change the style, follow these steps.

  1. Click STYLE in the left navigation.
  2. A Theme is a set of colors that can be applied to a template. Select the Light theme.
  3. To change a color, click on the associated color circle.
  4. Use the slider below the color square to select a color range.
  5. Click in the square to select a shade of the desired color. Color can also be set by entering the appropriate hex code, RGB value or HSV (hue, saturation, lightness) value into the text control.
  6. Click the < Back to Experience link at the top of the style menu.

To reset the color back to its original value, click the left arrow next to the color circle.

Publishing the experience

Mobile App experiences must be published before they can be accessed o mobile devices. For information on previewing and publishing a Mobile App experience, see Previewing and Publishing a Mobile App Experience.

To publish the experience, follow these steps.

  1. From the Experience Editor, click Publish.
  2. On the Publish this Mobile Application dialog, click Publish Now to confirm the publication.

Note the Access Key as it will be needed to access the experience on mobile devices.

Accessing the experience on a device

Once the Mobile App experience has been published, the Access Key is needed to access the experience on a mobile device.

To access the experience on a mobile device, follow these steps.

  1. If you don't have the Access Key, the Access Key is displayed as part of the project properties on the Projects homepage. Click on the experience to select it. The Access Key will be displayed in the properties section.
  2. Open the Brightcove Engage app on your device. Enter the Access Key and then click Next.
